## Risks + mitigations

**Risk 1:** GHL API rate limits during 3,171-contact extract
Mitigation: paginated extraction with delays, full extract may take hours, run overnight

**Risk 2:** Document storage from GHL hosted on their CDN, links could expire
Mitigation: extract documents to Supabase Storage during import, not just URLs

**Risk 3:** IQD unit mismatch (dinars vs fils) causes 1000x payment errors
Mitigation: Stage 1 verification of unit format, Stage 7 Saif validates payment amounts

**Risk 4:** Doctor tag has typos/variations (Saif / Dr. Saif / saif / etc.)
Mitigation: Stage 1 surfaces all tag variations, normalize during import, edge cases reviewed manually

**Risk 5:** Saif finds data fidelity issues in Stage 7, requires multiple re-imports
Mitigation: build for idempotent imports from start, separate test import vs production import code paths

**Risk 6:** Scope creep — Saif sees Velo, asks for new features not in V1.5
Mitigation: explicit V1.6 backlog, yes-but-not-in-V1.5 answer template, weekly progress updates

**Risk 7:** Vercel function ceiling hit while building Stage 4 automation
Mitigation: Pro upgrade done before Stage 4, function inventory documented
